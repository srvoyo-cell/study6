# Система Управления Окружающей Средой (Environmental Control System)

## Описание

Этот проект представляет собой симуляцию системы управления окружающей средой в помещении (например, в умном доме или офисе). Система анализирует входные данные от сенсоров (температура, влажность, уровень CO2, освещённость, наличие людей и время суток) и автоматически регулирует устройства: климат-контроль, увлажнитель, вентиляцию, основное и ночное освещение, а также систему оповещения.

Проект написан на Python и демонстрирует принципы объектно-ориентированного программирования, использование перечислений (enums), dataclasses и протоколов (protocols) для типизации. Логика контроллеров основана на простых условных правилах с пороговыми значениями.

Цель проекта — моделирование автономной системы, которая поддерживает комфортные условия и реагирует на критические ситуации (например, перегрев или высокий уровень CO2).

## Функции

- **Климат-контроль**: Регулирует нагрев и охлаждение на основе температуры и времени суток.
- **Увлажнитель**: Увеличивает влажность при низких значениях.
- **Вентиляция**: Управляет скоростью вентилятора в зависимости от уровня CO2 и наличия людей.
- **Основное освещение**: Регулирует яркость в зависимости от освещённости, наличия людей и времени суток.
- **Ночное освещение**: Включается ночью при низкой освещённости и наличии людей.
- **Система оповещения**: Выдаёт предупреждения или тревогу при критических значениях температуры или CO2.
- **Оркестрация**: Класс `ControlSystem` координирует обновление и вычисления для всех устройств.

## Установка и настройка

### Требования
- Python 3.8 или выше (используются фичи вроде `dataclasses` и `typing.Protocol`).

### Установка
1. Клонируйте репозиторий (если применимо) или скопируйте файлы в директорию `lab1`.
2. Убедитесь, что все файлы присутствуют:
   - `controllers.py`
   - `data_structures.py`
   - `main.py`
   - `main_controller.py`
   - `states.py`
3. Нет внешних зависимостей — проект работает на стандартной библиотеке Python.

## Использование

1. Запустите основной скрипт:
   ```
   python lab1/main.py
   ```
   Это симулирует три предопределённые конфигурации и выведет состояния устройств для каждой.

2. Пример вывода (для одной конфигурации):
   ```
   Состояние климат-контроля:
   Включен: ДА
   Нагрев: medium
   Охлаждение: off
   ----------------------------------------
   Состояние увлажнителя:
   Включен: НЕТ
   Режим: off
   ----------------------------------------
   # ... (для остальных устройств)
   ```

3. Чтобы протестировать с другими конфигурациями, отредактируйте `main.py` и создайте новые объекты `SystemConfig`.

### Пример конфигурации
В `data_structures.py` определён класс `SystemConfig`:
```python
config = SystemConfig(
    temperature=Temperature(15),
    humidity=Percent(50),
    co2=Ppm(1900),
    luminosity=Lux(400),
    people=True,
    time_of_day=Time.EVENING,
)
```

## Структура проекта

- **`data_structures.py`**: Определения типов, перечислений (Mode, Time, Alert) и класса конфигурации `SystemConfig`.
- **`states.py`**: Dataclasses для состояний каждого устройства (например, `ClimateControlStates`).
- **`controllers.py`**: Классы контроллеров с методами `update`, `compute` и `__str__` для каждого устройства.
- **`main_controller.py`**: Класс `ControlSystem` для оркестрации устройств и протокол `Controller`.
- **`main.py`**: Точка входа с примерами симуляций.
